### 📌 System Flow

```text
User → CDN → Load Balancer → Frontend / API Gateway → Auth Service / User Service / Music Service / Search Service / Playlist Service → Cache / Database / Queue → Worker / External Services
```

### 🧩 Service Breakdown

- **CDN** serves static assets fast
- **Load Balancer** distributes traffic
- **Frontend** handles UI and user interaction
- **API Gateway** routes requests to services
- **Auth Service** manages login, sessions, and authorization
- **User Service** manages profiles and account data
- **Music Service** handles tracks, metadata, and playback logic
- **Playlist Service** manages playlists and user collections
- **Search Service** handles discovery and indexing
- **Cache** stores hot data for faster reads
- **Database** stores persistent application data
- **Queue** handles async jobs like sync, notifications, and processing
- **Worker** processes background tasks
- **External Services** handle integrations such as storage, notification, and AI-related features

### 🎯 Design Goals

- **Scalable**: services can grow independently
- **Reliable**: async jobs are separated from the request flow
- **Maintainable**: each service has one clear responsibility
- **Fast UX**: cache and CDN reduce response time
- **Production-ready**: built with real system boundaries in mind
